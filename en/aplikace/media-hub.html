Home Media Hub

🎬Home Media Hub

My own tidy front-end over the movies, series and music stored on my home NAS. It runs entirely on my network — on a Raspberry Pi kiosk next to the TV, beside the smart-home dashboard. It can safely import scattered media from an external disk to the NAS and pull in artwork and descriptions from Jellyfin. No cloud, my library never leaves home.

Python (stdlib) SQLite Raspberry Pi kiosk Jellyfin LAN-only (no cloud)

← Back to My apps

What it does

My home NAS is full of movies, series, music and home videos — but clicking through network folders to find them is no fun. Media Hub is a light, nice front-end over it: it shows the library as tiles, lets me search and filter by type, mark items as "seen" and "favourite" — and runs right next to the TV on a small Raspberry Pi.

Read-only, nothing gets broken. The dashboard treats the media on the NAS strictly read-only — it only browses them (no deleting, moving or overwriting). The "seen/favourite" flags and the index live in its own local database (SQLite), never on the NAS.

Safe import from disk to NAS. When I find an old external disk with scattered media, Media Hub first only scans it, sorts files by type (movies, series, music, photos, home videos, documents) and shows a plan preview. Only after I confirm does it copy — and only new files: the source is never deleted, nothing on the NAS is overwritten, and every copy is verified. The order is always scan → proposal → preview → confirm → copy → verify.

Artwork and descriptions from Jellyfin. Optionally it connects to my home Jellyfin (running on the NAS) and adds posters, genre, length and description to films — still read-only, it writes nothing to Jellyfin or the NAS.

Tied into the smart home. On the kiosk by the TV one tap switches between the smart-home dashboard and Media Hub — two interfaces, one screen.

Key features

  • Library as tiles — movies, series, music, home videos and a separate Italian-films section; search by name, filter by type, "seen" and "favourite".
  • Read-only over the NAS — the only operation on the library is reading; flags and index in local SQLite, never on the NAS.
  • Safe import from an external disk — scans the source, sorts by type, shows a preview (dry-run) and copies only after confirmation; new files only, no overwrite, with verification (size + checksum).
  • Jellyfin integration — posters, genre, length and description from the home Jellyfin; display only, read-only.
  • Kiosk by the TV — Raspberry Pi in full-screen mode, touch switching between smart home and media, an automatic library board.
  • No cloud, LAN only — it all runs on my network, no account, the library never leaves home, a light core in pure Python.

Status

Work version v0.2.3 (June 2026). It runs at home on a Raspberry Pi kiosk by the TV, reading the library from the NAS. Private, home network (LAN) only, no public hosting and no installer to download — the real library and personal data never go on the web. Light core in pure Python (no dependencies), vanilla-JS front-end.
