⚙️Config Center

Control panel for the whole smart home — instead of one app per gadget, I have a single web page where I tune everything: when the morning routine starts, how warm the heating is, when audio stops, what lights do when I come home.

What it does

I have about 60 devices at home (lights, motion sensors, TRV heads, speakers, air purifier, blind, …) running on Homey Pro 2026. It can do a lot — has its own mobile app, web UI, scripts, automations — but there are far too many drawers and the settings are scattered.

Config Center unifies everything in one place. Open the web in a browser, log in with my token (the token stays only in this browser, nothing is sent anywhere), and see 22 clean pages: one per „house mode“ (morning / day / evening / night / away / home / sleep), separate for heating, lights, audio, etc.

Preview mode = safe test. When I want to try something, the app applies it first, gives me 10 minutes to test, and if I don't say „OK, save it“ in that window, it reverts itself. So even if I make a mistake, the house heals itself.

Protects from mistakes. Some devices (bedroom audio, sensitive sensors) have a hard-block — they cannot be turned on accidentally. The bedroom speaker can't play a test if someone's sleeping there.

Local, mine, no cloud. The app is just an HTML+CSS+JS file opened in Edge or Chrome. It talks directly to the Homey hub at home, nothing goes to a third-party server. No subscription, no vendor cloud dependency.

Key features

  • 22 pages per mode — Morning / day / evening / night / away / home / sleep — each with its own voice messages (TTS), light goals, heating goals.
  • Per-day, per-hour heating plan — For each of 4 zones (dining, bedroom, bathroom, toilet) day by day, hour by hour what temp I want. Boost button for quick heating.
  • 10-min preview safety — Change applies first, 10 min to test, auto-reverts if I don't confirm. Can't break the house with one typo.
  • 100+ knobs to tune — When morning starts, how fast lights ramp up, how long audio stays on, how loud the morning briefing is, …
  • Token in browser only — Hub access stored only in this browser's localStorage. No server, no clouds, no „sign in with Google“.
  • Hard-block for sensitive gear — Some devices can't be accidentally on. E.g. bedroom audio while sleeping, purifier during mold treatment, etc.

📖 Dev diary — Config Center

  • v1.8 — AI check (Gemini), May 30, 2026 — A new “🧠 AI check” page right inside Config Center. A safe AI auditor: reads all settings and looks for conflicts, suspicious ranges, mismatches with reality and misleading help texts. The AI changes nothing — it only analyzes, explains and recommends.
  • v1.7 — Token form + cleanup (May 2026) — Token login via a form (token only in the browser, localStorage), connection health indicators, preview mode with a 10-min auto-revert.
  • CC-FINAL — 35 sh_cfg_* variables (May 24–27, 2026) — Unification: Config Center is the single place for persistent settings, the dashboard only controls. A universal guard prevents double writes.
  • v1.7.5 (May 2026) — 100+ live-editable smart-home variables. Runs on the PC and on the wall panel (RPi kiosk).
  • Split from the dashboard — Config Center = persistent settings (sh_cfg_*), dashboard = runtime control. No mixing.
  • What it configures — Thresholds (lux, temperatures), morning-routine times, dim ramps, AI toggles, tariffs. All as Homey Logic variables.
