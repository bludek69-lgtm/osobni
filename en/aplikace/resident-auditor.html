Resident Auditor

🔎Smart Home Resident Auditor

A tool that judges my smart home through the eyes of the person living in it — not „the script ran“, but „did the morning actually work the way it should?“. It reads the real logs for a given day and returns PASS / WATCH / FAIL with evidence. Run manually, changes nothing in the home.

What it does

My home is run by lots of automations — lights, audio, heating, blinds, scenes. Ordinary tools only report „the script finished without error“. But a technically successful script doesn't mean it was pleasant for a human. The radio might start three times in a row, a light might switch on at full brightness at night, the heating might run while nobody is home — and all of it „technically ran OK“.

Resident Auditor looks differently. It takes the stored logs of the home for a specific day and asks, in human terms: Was the morning calm? Did the lights react to movement and did the blinds take priority? Did anything stay on or keep playing after I left? Did the heating behave sensibly? Did anything wake me at night?

The key rule: never PASS without evidence. If there's no proof something went well, it doesn't get a green light — it gets „WATCH“. And the reverse too: routine, harmless events don't count as a problem, so there are no false alarms. The goal is a trustworthy verdict, not pretty statistics.

It changes nothing. The tool is strictly read-only — it only reads logs. It never touches the home, fixes nothing, and doesn't run in the background. I launch it manually when I want to know how the home behaved.

Key features

  • 10 areas of daily life — morning, night movement, audio & voice messages, lights & blinds, bathroom, heating, device reliability and system stability. Each has its own „auditor“.
  • Evidence-based verdict: PASS / WATCH / FAIL — green only with positive proof of correct behaviour; otherwise watch; red only for a real problem.
  • No false PASS and no false alarm — it tells a real action apart from a routine check, so it doesn't flag a problem where there is none.
  • A suggested next step for every problem — when something fails, a „task to solve“ is created with evidence (time, what happened, impact on the resident) — but a fix never runs on its own, it waits for my decision.
  • Audit of a specific day — I run it for a chosen day and get a clear report; ideal as a check after every change to the home.
  • Manual, safe — no automatic scheduling, no interference with the home, read-only.

Sample output

Audit of one real day (shortened):

$ python -m core.runner --run --real --day 2026-06-02

  morning (wake-up)     : PASS   — calm start, briefing once, no disturbance
  night movement        : PASS   — handled dimly, nothing woke me
  audio / radio / TTS   : PASS   — radio at most once, no night announcements
  lights + blinds       : PASS   — lights by scene, blinds took priority
  bathroom / toilet     : PASS   — light by phase, guardian idle
  heating & comfort     : PASS   — controlled by schedule, no waste
  system stability      : PASS   — 0 crashes
  dashboard vs reality  : PASS   — matches reality
  device reliability    : WATCH  — 1 non-critical device drops out now and then

  Overall: WATCH  ·  0 actions required  ·  0 false alarms

The verdict is WATCH, not „all green“ — because one device drops out occasionally. Honest, without sugar-coating.

Status

Version v0.1 — a finished foundation, ready for manual use (June 2026). Verified on real days, 72 automated tests pass. Add-ons (a daily „story of the home“, finer device classification) are planned, not required.

📖 Dev diary — Resident Auditor

  • v0.1 done — calibrated on real data (June 2026) — The tool reads the home's real logs and, for lights, bathroom, heating and the morning, gives PASS/FAIL based on actual actions (not the number of log lines). Verified on two real days: 9 areas green with evidence, 1 to watch — 0 false alarms.
  • Fixing false alarms — It turned out routine „check“ events (the guardian verified something, all OK) must not count as a problem. A „semantic layer“ now tells a real action from a routine one — so the tool doesn't raise an alarm where there's nothing.
  • Connecting to the home's real logs — The home has been storing detailed behaviour records for a while. The tool connected to them (read-only) and learned to interpret them correctly — including time zones and context (time of day, presence, brightness).
  • Ten auditors (plugins) — Areas were added gradually: morning and system crashes, then audio/lights/bathroom, then heating/night/dashboard/devices. Each is independent and testable.
  • The core principle from day one — Never PASS without evidence. Technical success is not enough. Read-only, run manually, never touches the home without my approval.
