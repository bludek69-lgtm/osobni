App Tester

🧪Universal App Tester

A tool that walks through an entire web app on its own: it launches a browser, opens every page and returns PASS / WATCH / FAIL — did something fail to load? are there console errors? is the page empty or slow? Adding a new app = one entry in a config file. Same controls as the Resident Auditor.

Python Playwright Read-only

What it does

I build my own apps — a meal planner, an Italy travel planner, an accounting book, a collection catalogue, … After every change I need to check that all the pages still load normally, nothing throws in the console and nothing is broken. Clicking through it by hand is tedious and easy to miss something.

App Tester does it for me. It launches a browser in the background (no window), opens each page of the chosen app one by one and records for each: did it load at all? how fast? are there errors in the browser console? does the page have any content, or is it empty? do the pages differ from each other (does routing work)? From that it composes a clear verdict.

Universal = no hardcoding. Apps aren't baked into the code — they live in a simple list (config). Adding a new app means writing a few lines: address, list of pages. The tester is generic, it works on anything that runs in a browser.

Same philosophy as the Resident Auditor: never PASS without evidence. When an app isn't running, it gets neither green nor red — it gets „WATCH“ (not enough evidence), so no falsely good result is produced. And the tester changes nothing — it only looks (read-only), runs manually, doesn't run in the background.

Key features

  • 6 checks per page — page load, console errors, empty content, load speed, and whether the pages differ from each other (routing works).
  • PASS / WATCH / FAIL verdict — FAIL when something doesn't load or the console throws; WATCH when a page is slow, empty or the app isn't running; PASS only when everything is fine.
  • Universal config — adding an app = one object with a list of pages. No code rewriting.
  • Screenshots + report — a snapshot is saved from each page; the result is a clear per-app report plus a summary across all of them.
  • No false PASS — when an app isn't running it's honestly WATCH, not falsely green.
  • Manual, read-only — no automatic scheduling, it only looks, deploys nothing.

Sample output

First a self-test (verifies everything is ready), then a test of one app:

$ python -m core.runner --self-test
  [PASS] mode: run on demand
  [PASS] browser + Playwright available
  [PASS] check: app not running → WATCH
  [PASS] check: console error → FAIL
  [PASS] check: clean run → PASS
  SELF-TEST RESULT: PASS

$ python -m core.runner --run --app meal-planner
  meal-planner: FAIL — 6/7 pages loaded, 1 console error
  Overall: FAIL · report and screenshots saved

The tester reports the concrete problem (which page, what error) — not just „something went wrong“.

Status

Version v0.1 (June 2026). Self-test passes (12/12), verified live on a real app. It grew out of earlier one-off scripts, which I unified into a single universal tool controlled the same way as the Resident Auditor.
