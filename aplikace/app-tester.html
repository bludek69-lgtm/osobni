App Tester

🧪Univerzální tester aplikací

Nástroj, který sám projde celou webovou aplikaci: spustí prohlížeč, otevře všechny stránky a řekne PASS / WATCH / FAIL — nenačetlo se něco? jsou chyby v konzoli? je stránka prázdná nebo pomalá? Přidání nové aplikace = jeden řádek v configu. Stejné ovládání jako Resident Auditor.

Co to dělá

Dělám si vlastní aplikace — plánovač jídel, cestovní plánovač Itálie, účetní knihu, sbírku, … Po každé úpravě je potřeba zkontrolovat, že se všechny stránky pořád normálně načtou, nikde nepadá konzole a nic není rozbité. Proklikávat to ručně je otrava a snadno se na něco zapomene.

App Tester to udělá za mě. Spustí na pozadí prohlížeč (bez okna), postupně otevře každou stránku zvolené aplikace a u každé si zapíše: načetla se vůbec? jak rychle? jsou tam chyby v konzoli prohlížeče? má stránka nějaký obsah, nebo je prázdná? liší se stránky mezi sebou (funguje přepínání)? Z toho složí přehledný verdikt.

Univerzální = žádný hardcode. Aplikace nejsou napevno v kódu — jsou v jednoduchém seznamu (config). Přidat novou appku znamená dopsat pár řádků: adresa, seznam stránek. Tester je obecný, funguje na cokoliv, co běží v prohlížeči.

Stejná filozofie jako Resident Auditor: bez důkazů nikdy PASS. Když aplikace zrovna neběží, nedostane zelenou ani červenou — dostane „WATCH“ (nedostatek důkazů), ať se nevyrobí falešně dobrý výsledek. A tester nic nemění — jen se dívá (read-only), spouští se ručně, neběží na pozadí.

Klíčové funkce

  • 6 kontrol u každé stránky — načtení stránky, chyby v konzoli, prázdný obsah, rychlost načítání, a zda se stránky navzájem liší (funguje routing).
  • Verdikt PASS / WATCH / FAIL — FAIL když se něco nenačte nebo padá konzole; WATCH když je stránka pomalá, prázdná nebo appka neběží; PASS jen když je vše v pořádku.
  • Univerzální config — přidání aplikace = jeden objekt se seznamem stránek. Žádné přepisování kódu.
  • Screenshoty + report — z každé stránky se uloží snímek; výsledek je přehledný report po aplikacích i souhrn za všechny.
  • Bez falešného PASS — když appka neběží, je to poctivě WATCH, ne falešně zelená.
  • Ručně, read-only — žádné automatické spouštění, jen se dívá, nic nenasazuje.

Ukázka výstupu

Nejdřív sebe-test (ověří, že je vše připravené), pak test jedné aplikace:

$ python -m core.runner --self-test
  [PASS] režim: ručně spouštěné
  [PASS] prohlížeč + Playwright dostupné
  [PASS] kontrola: aplikace neběží → WATCH
  [PASS] kontrola: chyba v konzoli → FAIL
  [PASS] kontrola: čistý běh → PASS
  VÝSLEDEK SELF-TEST: PASS

$ python -m core.runner --run --app meal-planner
  meal-planner: FAIL — 6/7 stránek načteno, 1 chyba v konzoli
  Celkový stav: FAIL · report a screenshoty uloženy

Tester nahlásí konkrétní problém (která stránka, jaká chyba) — ne jen „něco se pokazilo“.
